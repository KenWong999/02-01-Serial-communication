# ESP32 串口控制 LED 亮灭实验

## 一、项目简介

本项目使用 ESP32-WROOM-32 开发板，通过串口通信方式接收电脑串口助手发送的字符串指令，并根据指令控制外接 LED 的亮灭。

实验中，单片机接收到串口助手发送的字符串后，会对字符串进行解析，并与预设指令进行对比。若指令有效，则切换对应 LED 的亮灭状态；若指令无效，则不执行动作。同时，ESP32 会通过串口向电脑返回执行结果，便于在串口助手控制台中观察反馈信息。

## 二、实验器材

* ESP32-WROOM-32 开发板 × 1
* LED × 3
* 220Ω 或 330Ω 限流电阻 × 3
* 面包板 × 1
* 杜邦线若干
* USB 数据线 × 1
* 串口助手软件 × 1

## 三、接线说明

每颗 LED 都需要串联一个限流电阻，防止电流过大损坏 LED 或开发板 IO 口。

接线方式如下：

```text
LED1：GPIO4  → 220Ω电阻 → LED长脚 → LED短脚 → GND
LED2：GPIO16 → 220Ω电阻 → LED长脚 → LED短脚 → GND
LED3：GPIO17 → 220Ω电阻 → LED长脚 → LED短脚 → GND
```

LED 长脚为正极，连接 GPIO 输出端；LED 短脚为负极，连接 GND。

## 四、串口通信设置

串口助手参数设置如下：

```text
波特率：115200
数据位：8
停止位：1
校验位：无
发送格式：字符串 / 文本
```

## 五、控制指令说明

| 串口发送字符串 | 功能            |
| ------- | ------------- |
| `1`     | 切换 LED1 的亮灭状态 |
| `2`     | 切换 LED2 的亮灭状态 |
| `3`     | 切换 LED3 的亮灭状态 |
| `4`     | 无对应指令，不执行动作   |

本实验中特别注意：串口助手发送的是字符串，例如 `"1"`，程序中解析和比较的也是字符串，而不是整数 `1`。

## 六、实验现象

当串口助手发送字符串 `1` 时：

* 如果 LED1 原来是熄灭状态，则 LED1 点亮；
* 如果 LED1 原来是点亮状态，则 LED1 熄灭；
* ESP32 通过串口返回当前执行结果，例如 `LED1亮` 或 `LED1灭`。

当串口助手发送字符串 `2` 时，LED2 的状态发生切换。

当串口助手发送字符串 `3` 时，LED3 的状态发生切换。

当串口助手发送字符串 `4` 或其他无效指令时，LED 状态不变，串口控制台显示：

```text
无对应指令，不执行动作
```

## 七、核心代码逻辑

程序主要分为三个部分：

1. 初始化 LED 引脚和串口通信；
2. 在 `loop()` 函数中不断检测串口是否收到数据；
3. 将接收到的字符串与预设指令进行比较，并控制对应 LED 状态。

关键判断逻辑如下：

```cpp
if (command == "1") {
  led1State = !led1State;
  digitalWrite(LED1_PIN, led1State ? HIGH : LOW);
}
```

其中，`led1State = !led1State;` 表示将 LED1 的当前状态取反，从而实现“发送一次指令，状态切换一次”的效果。
